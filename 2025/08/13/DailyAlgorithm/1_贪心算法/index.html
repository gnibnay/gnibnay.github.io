贪心算法—局部最优的策略

Last updated on August 14, 2025 pm

贪心思想

  之所以称为贪心是因为在算法的某个状态，在无法获知全局信息的情况下选择当前状态下能够带来最大“收益”的操作，即局部最优的选择。在一些问题中，保证每次都选择局部最优的操作，最终能够导向全局最优的结果，这就是贪心思想。

但什么样的问题适合用贪心思想来解决？

  适合用贪心算法来解决的问题具有两个特点：贪心选择性质和最优子结构。

1.贪心选择性质

  所谓的贪心选择性质就是说：问题的全局最优解可以通过一系列的局部最优解来构建。每一步都是当前状态下的最优选择，从而能够最终构成全局状态下的最优解。需要注意的是，可以由局部最优导向全局最优，但这可能并非唯一的解路径。

2.最优子结构

  最优子结构体现了问题的一种特殊性质：问题的最优解包含了其子问题的最优解。也就是说，如果问题的解可以分解成若干子问题的解，那么这些子问题的解也时最优的。

上述两个性质看起来很显然，但仍需要结合实际问题进行分析。

No.455 分发饼干

1.问题描述

  假设你是一位很棒的家长，想要给你的孩子们一些小饼干。但是，每个孩子最多只能给一块饼干。对每个孩子 $i$，都有一个胃口值 $g[i]$，这是能让孩子们满足胃口的饼干的最小尺寸；并且每块饼干 $j$，都有一个尺寸 $s[j]$ 。如果 $s[j] >= g[i]$，我们可以将这个饼干 $j$ 分配给孩子 $i$ ，这个孩子会得到满足。你的目标是满足尽可能多的孩子，并输出这个最大数值。

2.输入输出样例

输入: g = [1,2,3], s = [1,1]
输出: 1
解释:
  你有三个孩子和两块小饼干，3 个孩子的胃口值分别是：1,2,3。
虽然你有两块小饼干，由于他们的尺寸都是 1，你只能让胃口值是 1 的孩子满足。
所以你应该输出 1。

3.题目分析

为什么要选择贪心策略，以及选择什么样的贪心策略？

  饼干分发是一个典型的分配问题，为了让更多的孩子得到满足，直观的想法是每次分配应尽可能地留下更多的资源，避免不必要的浪费。若将其作为问题解决的策略，就不得不考量是否能够得到最终的最优解。很明显，这个策略符合典型的贪心思想的特点，即优先选择最小尺寸的饼干去满足当前胃口最小的孩子。对于贪心策略需要验证其是否满足两大性质——贪心选择性质和最优子结构，这是确保能够到达全局最优的基础。

是否满足贪心选择性质

  贪心选择性质要求问题的最优解可以由一系列的局部最优解构建而来。我们需要证明，通过每一步的局部最优选择，最终能够构建出全局最优解。

  假设有这样的一组输入：

  • 孩子的胃口值数组：$$g=[g_1, g_2, …, g_n]$$ 其中 $g_1 \le g_2 \le … \le g_n$；
  • 饼干的尺寸大小数组：$$s=[s_1, s_2, …, s_m]$$ 其中 $s_1 \le s_2 \le … \le s_m$.

  同时，我们假设存在一个最优解$OPT$，在这个最优解中，某个孩子$g_k$（$1 \le k \le n$）被分配了饼干$s_l$($1 \le l \le m$)，其中$s_l$并非是最小的能够满足孩子$g_k$的饼干，即至少存在一个饼干$s_j$，使得$g_k \le s_j \le s_l$（$1 \le j \le l \le m$）。

  那么我们就可以构造一个新的解$OPT^{‘}$，将饼干$s_j$分配给孩子$g_k$，而可以将更大一些的饼干$s_l$分配给其他胃口更大的孩子。那么饼干$s_l$是否能够满足其他的孩子呢，我们并不关心，因为至少$OPT^{’}$不会比$OPT$更劣，即若$OPT$确实是一个全局的最优解，那么$OPT^{'}$同样也是。

  于是我们也证明了对于一个全局的最优解，通过一系列的局部最优解同样可以得到相同的结果，即能够由局部最优导向全局最优。

是否满足最优子结构

  最优子结构要求问题的最优解可以有其子问题的最优解组成。最优子结构的证明实际上隐含在了贪心选择性质的证明中，贪心选择性质和最优子结构逻辑上紧密相连，但是侧重点不同。前者证明局部最优选择的存在性以及局部最优选择导向全局最优解，而后者则证明子问题的独立性和子问题的组合性。

  对于饼干分配问题，每一次为孩子分配饼干的过程是独立的，这里的独立并非说前后两次的分配之间没有任何影响，而是说前一次的分配并不会影响下一次采用局部最优策略的分配过程，即每个子问题的最优解不会因为前面的局部最优选择而变得无效。而前面证明的通过一系列的局部最优能够最终构建全局最优解恰恰证明了子问题的组合性，即子问题的最优解组成了问题的最优解。

4.题解

  经过上述分析，我们可以得到饼干分配问题的解题思路：将孩子的胃口值数组$g$和饼干尺寸大小数组$s$分别按升序排序，然后依次对每个$g_i$选择最小的$s_j$，使得 $g_i \le s_j$，直到$g$或$s$遍历结束。

  这里可以使用双指针的技巧，一个指针指向数组$g$中的元素，表述当前需要满足的孩子$g_i$，而另一个指针指向数组$s$中的元素，表示最小的能够满足孩子$g_i$的饼干$s_j$。这里的两个指针$i$和$j$都不会回溯，因为连胃口更小的孩子都满足不了的饼干必然无法满足胃口更大的孩子。
